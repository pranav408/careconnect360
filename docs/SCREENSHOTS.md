# Screenshot Guide

## Purpose
This guide defines what to capture for portfolio, GitHub documentation, and interview walkthroughs.

## Storage Location
- Recommended folder: `docs/screenshots/`

## Naming Convention
- `01-login.png`
- `02-patient-dashboard.png`
- `03-doctor-directory.png`
- `04-patient-appointments.png`
- `05-patient-insurance.png`
- `06-patient-claims.png`
- `07-patient-payments.png`
- `08-patient-notifications.png`
- `09-doctor-appointments.png`
- `10-doctor-profile.png`
- `11-admin-dashboard.png`
- `12-admin-doctors.png`
- `13-admin-insurance.png`
- `14-admin-claims.png`

## Capture Checklist
- Login screen and role-aware post-login redirects.
- Patient dashboard summary state.
- Doctor directory with search/filter context.
- Patient appointment list (including status badges).
- Insurance list with at least one ACTIVE policy example.
- Claim list showing lifecycle states.
- Payment history including one SUCCESS row.
- Notifications list including read/unread state.
- Doctor appointment workspace showing transition actions.
- Doctor self-profile screen.
- Admin dashboard aggregates.
- Admin doctor creation view.
- Admin insurance decision queue.
- Admin claim decision queue.

## Privacy and Security Rules
- Do not include passwords.
- Do not include JWT tokens.
- Do not include browser developer-console secrets.
- Do not include real personal information.
- Use demo/sample records and placeholder identities.
- Crop unnecessary browser UI when appropriate.

## Quality Tips
- Keep consistent browser zoom.
- Capture full relevant table/panel context.
- Prefer clean states with readable status transitions.
- Do not generate synthetic or fake screenshots.
